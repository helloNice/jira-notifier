# 小青龙 - Jira 待办提醒

![封面图](https://github.com/jiangjiji/jira-notifier/blob/main/doc/img/cover.png?raw=true)

小青龙 - Jira 待办提醒是一个 Chrome 扩展，用于定时检查当前登录用户在 Jira 中未解决且指派给自己的任务，并在后台轮询发现新任务或重新打开任务时发送通知。

## 功能概览

- 首次安装后自动打开 Jira 地址配置页，不需要先点击扩展里的“登录”。
- 首次配置页只要求输入 Jira 服务器地址，并通过 Chrome host permission 授权访问该 Jira。
- 扩展复用当前 Chrome 中的 Jira 登录 Session，不保存 Jira 密码。
- 支持中文和英文界面，语言跟随 Chrome 扩展环境。
- 后台按间隔轮询当前 Jira 用户被指派的未解决任务，并对新增任务发送系统通知。
- popup 中展示当前任务列表、下一次检查倒计时和任务数量角标。
- 任务按 Jira 项目分组展示，支持拖动项目调整展示顺序。
- 每条任务提供复制按钮，可复制 `Jira Key + 任务标题`，例如 `TSXK-5843 吞噬星空卡牌战斗模拟测试接入`。
- 支持按 Jira `duedate` 字段做到期提醒，可配置 1 天、5 小时、1 小时、15 分钟前提醒。
- 支持自定义后台轮询使用的 Jira JQL，并可在设置页先校验 JQL 是否可用。

## 默认配置

扩展默认配置保存在浏览器本地存储中：

- Jira 地址：首次安装后会自动打开专用配置页，例如填写 `https://jira.example.com`
- 开启检测：默认开启
- 检测间隔：默认 `180` 秒，设置页可在 `60` 到 `600` 秒之间调整
- 通知方式：默认“系统通知”
- 不通知：关闭系统通知，仅更新 popup 中的任务列表和角标
- 直接跳转：默认开启；开启后，从 popup 点击任务会打开 Jira 任务页并自动切换过去
- 到期提醒：默认关闭；开启后可选择 `1天前`、`5小时前`、`1小时前`、`15分钟前`
- Jira JQL：默认 `resolution = Unresolved AND assignee in (currentUser()) ORDER BY updated DESC`

设置页修改会立即写入本地存储；填写 Jira 地址并授权后，后台会按检测间隔自动轮询。

## 首次安装与登录

1. 安装扩展后，会自动打开“小青龙 Jira 待办提醒”的连接页面。
2. 输入 Jira 首页或任意 Jira 页面链接，例如 `https://jira.example.com`。
3. 点击“授权并保存”，Chrome 会请求该 Jira 地址的访问权限。
4. 授权成功后会打开 Jira 登录页。
5. 在同一个 Chrome 中完成 Jira 登录后，再打开扩展即可查看任务。

该流程只保存 Jira 服务器地址，不保存 Jira 用户名或密码。

## 通知规则

### 会触发通知

- 后台自动轮询发现此前没有通知过的新未解决任务时，会触发通知。
- 设置页点击“测试通知”会触发一条测试通知；如果通知方式为“不通知”，测试按钮只会提示当前无法测试，不会创建系统通知。

### 不会触发通知

以下操作只刷新或整理本地任务数据，不创建系统通知：

- 打开扩展 popup 时的自动刷新。
- popup 任务页显示“距离下次检查”的倒计时。
- 设置页点击“重置记录”后立即重新拉取任务。
- 设置页点击“全部隐藏”。
- 在 popup 中点击某个任务后，该任务会加入隐藏列表并从当前列表移除。
- 已经通知过、仍停留在未解决列表中的同一个任务，不会在后续轮询中重复通知。
- 通知方式设置为“不通知”时，后台轮询不会创建系统通知。

## 多任务通知与点击行为

一次后台轮询如果发现多个需要通知的新任务，只会聚合成一条系统通知：

- 通知标题：`新任务指派（N个）`
- 通知正文：`包括「<第一个任务Key> <第一个任务标题>」在内的 N 个新任务`

如果只发现一个任务，正文直接显示该任务的 Key 和标题。

点击通知本体或通知按钮会打开通知中第一个任务的 Jira 详情页；如果找不到任务映射，则打开配置的 Jira 首页。

## 到期提醒

到期提醒读取 Jira issue 的 `duedate` 字段。设置页可开启到期提醒，并选择以下提醒时间：

- `1天前`
- `5小时前`
- `1小时前`
- `15分钟前`

后台轮询时，如果任务距离截止时间已经进入某个提醒窗口，会发送到期提醒通知。多个任务命中同一个提醒窗口时会聚合成一条通知，例如“包括「<第一个任务Key> <第一个任务标题>」在内的 N 个任务即将在 1小时内到期”。

Jira 的 `duedate` 通常是日期型字段。如果只有日期没有具体时间，扩展会复用任务创建时间的时分秒来计算提醒时间；如果无法取得创建时间，则按当天结束前处理。

同一个任务、同一个截止时间、同一个提醒窗口只提醒一次；如果 Jira 中的截止时间发生变化，会重新计算提醒记录。

## Reopen / 重新打开任务再次提示

后台轮询检查的是 Jira JQL：

```text
resolution = Unresolved AND assignee in (currentUser()) ORDER BY updated DESC
```

因此 Reopen 后重新进入未解决列表且指派给当前用户的任务也会出现在检查结果中。

当前规则是：当任务从未解决列表消失后，扩展会清除该任务的已通知记录；如果之后该任务重新进入未解决列表，下一次后台自动轮询会把它当作新的待提示任务再次通知。

## 任务列表操作

- 点击任务：打开该任务的 Jira 详情页；如果“直接跳转”开启，会自动切换到新打开的 Jira 页面。
- 复制任务：点击任务右侧复制按钮，复制内容为 `Jira Key + 任务标题`。
- 项目排序：拖动项目标题左侧的拖拽手柄，可以调整项目分组在 popup 中的展示顺序。
- 下一次检查：任务页顶部显示后台下一次自动检查的倒计时；检测关闭时显示“检测已关闭”。

复制功能优先使用浏览器 Clipboard API；如果当前扩展环境不支持或调用失败，会降级使用隐藏文本框和 `document.execCommand("copy")`。

## 自定义 JQL

高级选项中可以修改后台轮询使用的 Jira JQL。留空时使用默认查询：

```text
resolution = Unresolved AND assignee in (currentUser()) ORDER BY updated DESC
```

点击“检测”会用当前 Jira 连接轻量请求 1 条结果来校验 JQL。校验通过后，扩展会保存该 JQL、清空本地隐藏/已通知记录，并重新拉取当前任务；这次重新拉取不会触发新任务通知。

## 数据管理

设置页底部提供两个数据管理按钮：

- 全部隐藏：把当前列表中的所有任务加入隐藏列表，立即从 popup 列表移除，并且不对这些任务再次发送通知。
- 重置记录：清空隐藏记录和已通知记录，然后重新拉取当前未解决任务。该次拉取不会发送通知，用于恢复列表展示和重新整理本地记录。

## 构建与打包

常用命令：

```bash
npm run compile      # TypeScript 类型检查
npm run build        # 构建 Chrome MV3 扩展到 .output/chrome-mv3
npm run zip          # 打包 Chrome 扩展
```

Windows 下可使用 `build-packages.bat` 一次完成类型检查和打包：

```bat
build-packages.bat
build-packages.bat 1.3.1
```

传入版本号时，脚本会同步更新 `package.json` 和 `package-lock.json` 中的版本号。脚本当前会打包 Chrome 和 Edge；Firefox 打包已暂时关闭，因为当前 Firefox 构建未通过校验。
