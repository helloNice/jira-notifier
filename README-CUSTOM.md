# Jira 任务通知助手（精简版）

基于 [jiangjiji/jira-notifier](https://github.com/jiangjiji/jira-notifier) 改造的精简版。

## 功能

- **自动检测**：默认每隔 3 分钟轮询一次，检测指派给你的 Jira 未解决任务
- **桌面通知**：后台自动轮询发现新任务或重新打开任务时，弹出系统通知（可点击跳转）
- **倒计时显示**：任务页显示距离下一次自动检查还剩多少秒，不再提供手动刷新按钮
- **忽略管理**：可隐藏单个或全部任务，支持重置记录
- **夜间安静**：默认 180s 轮询间隔，对 Jira 服务器压力可忽略

## 安装步骤

### 1. 加载扩展

1. 打开 Chrome，访问 `chrome://extensions/`
2. 开启右上角 **「开发者模式」**
3. 点击 **「加载已解压的扩展程序」**
4. 选择目录：`D:\workbuddy\jirame\jira-notifier-lite\.output\chrome-mv3`

### 2. 登录 Jira

1. 点击扩展右上角设置按钮，填写你的 Jira 服务器地址，例如 `https://jira.example.com`
2. 在同一个 Chrome 中打开该 Jira 地址
3. 登录你的 Jira 账号（扩展会自动复用浏览器的登录 Session）
4. 点击浏览器工具栏的扩展图标，应看到你的任务列表

### 3. 验证通知

- 后台自动轮询发现此前未通知过的未解决任务时，桌面会弹出系统通知
- 已解决/已关闭任务从未解决列表消失后，会清除已通知记录；如果验收未通过等原因被重新打开，下一轮后台自动检查会再次提示
- 打开扩展弹窗、查看倒计时、重置记录、全部隐藏等操作不会触发系统通知
- 如需修改轮询间隔或通知方式，点击弹窗右上角的 ⚙ 设置

## 改造内容

| 改动 | 说明 |
|------|------|
| Jira 服务器地址 | 不预设，首次使用需要在设置页手动填写 |
| 轮询间隔 | 默认 **180 秒（3 分钟）**，可自定义（60-600s） |
| 删除死代码 | 移除未使用的 `AllBugLayout`，精简为单一任务列表 |
| 倒计时 | 任务页显示距离下一次后台自动检查的剩余秒数 |
| 通知修复 | `iconUrl` 使用 `browser.runtime.getURL()` 正确路径，确保通知不静默失败 |
| 通知策略 | 仅后台自动轮询和测试通知触发系统通知，popup 自动刷新/重置/隐藏不通知 |
| 重新打开 | 任务解决后从未解决列表消失会清除已通知记录，重新打开后再次通知 |
| 状态判定 | 移除对魔法 status.id 的依赖，新任务和重新打开任务统一分类为"新任务指派" |
| i18n 新增 | 添加通知标题、多任务聚合文案、倒计时等中英文提示文本 |

## 技术栈

WXT + React + TypeScript + antd + zustand + jira.js
